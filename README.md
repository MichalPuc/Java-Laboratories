
# Java Programming

## Lab01

Write an application that allows checking specified directories for changes in their files. The changes should be verified by calculating the MD5 hash function.

## Lab02

Create an application that enables browsing measurement data stored on the disk in CSV files.

## Lab03

Develop an application that consumes data obtained from a public RESTful API service.

## Lab04

Build an application that enables task execution by instances of classes loaded with a custom class loader.

## Lab05

Implement an application with a graphical interface to perform statistical analysis of classification results stored in confusion matrices.

## Lab06

Write a program that simulates the operation of a customer service tool for an Internet service provider. The program should utilize a relational database.

## Lab07

Transform the application developed in the previous lab into a network service.

## Lab08

Java 2D Convolution Function

Create a program using JNI to calculate the 2D discrete convolution function based on a declared Java class. The program requires two methods: one native and the other implemented in Java. 

## Lab09

XML Document Processing

During this laboratory, you will practice different ways of processing XML documents. Construct a program that:

Loads XML data using JAXB and displays it on the screen.
Loads XML data using JAXP and displays it on the screen (using SAX and DOM parsers).
Processes data from the XML document using an XSLT transformation stylesheet and displays the result of this transformation.

## Lab10

File Encryption

Implement an application that allows encrypting and decrypting files. The application can serve as a tool for encrypting and decrypting email attachments. The program should have a graphical interface that allows users to select input and output files, choose encryption/decryption algorithms, and provide the necessary keys (private key for encryption and public key for decryption). The encryption/decryption logic should be contained in a separate JAR library, which will be digitally signed. The application should be exported as an executable JAR file with a digital signature.

## Lab11

Based on the code from previous labs, prepare a multi-release JAR.

## Lab12
Dynamic Extension with JavaScript

The purpose of this laboratory is to practice dynamic extension of Java program functionality by loading and unloading JavaScript scripts (similar to loading classes with a custom class loader).

## Lab13
JavaFX Application using JavaScript Scripts

The goal of this laboratory is to implement an application that uses JavaFX in an unconventional way. All the application logic will be embedded in an FXML file rather than in a controller written in Java. The application function will involve generating entries for a journal based on previously prepared templates, which will be completed with the necessary attributes (date, name, etc.). The generated entries will be displayed on the user interface as text, which can be copied. The templates will be stored in configuration files.

## Lab09

Przetwarzanie dokumentów XML

Podczas tego laboratorium poćwiczysz różne sposoby przetwarzania dokumentów XML. Skonstruuj program, który:
Wczyta dane XML przy użyciu JAXB i wyświetli je na ekranie.
Wczyta dane XML przy użyciu JAXP i wyświetli je na ekranie (używając parserów SaX i DOM).
Przetworzy dane z dokumentu XML za pomocą arkusza transformacji XSLT i wyświetli wynik tego przetwarzania.

## Lab10

Szyfrowanie plików

Zaimplementuj aplikację umożliwiającą szyfrowanie i deszyfrowanie plików. Aplikacja może pełnić rolę narzędzia do szyfrowania i odszyfrowywania załączników do e-maili. Program powinien posiadać interfejs graficzny, który pozwala użytkownikowi wskazać pliki wejściowe i wyjściowe, a także wybrać algorytm szyfrowania/deszyfrowania oraz odpowiednie klucze (prywatny do szyfrowania i publiczny do deszyfrowania). Logika szyfrowania/deszyfrowania powinna być zawarta w osobnej bibliotece JAR, która zostanie podpisana cyfrowo. Aplikację należy wyeksportować jako wykonywalny plik JAR z podpisem cyfrowym.

## Lab13

Tworzenie aplikacji JavaFX z użyciem skryptów JavaScript

Celem tego laboratorium jest zaimplementowanie aplikacji, która używa JavaFX w niekonwencjonalny sposób. Cała logika aplikacji będzie osadzona w pliku FXML, a nie w kontrolerze napisanym w języku Java. Funkcja aplikacji będzie polegała na generowaniu wpisów do pamiętnika na podstawie wcześniej przygotowanych szablonów, uzupełnianych niezbędnymi atrybutami (data, imię itp.). Wygenerowane wpisy będą widoczne na interfejsie użytkownika jako tekst, który można skopiować. Szablony będą przechowywane w plikach konfiguracyjnych.
